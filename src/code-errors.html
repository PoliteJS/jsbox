JSBox Errors Handling

If you use JSBox to create some Javascript exercises or evaluation tests then you need to handle the situation in which a student write code... that just makes no sense! 

var

Prevoius code makes no sense but a total beginner doesn't understand that!

JSBox is able to detect such situation and alert with an error message.

Synchronous Bad Error:

The following box is pre-compiled with an horrible error that can not be detected by a normal try {} catch (e) {}: 

var
console.log("synchronous stuff has finished");

Asynchronous Bad Error:

The following box is pre-compiled with an horrible error that can not be detected by a normal try {} catch (e) {}.

This time the error is inside an async callback but it is detected anyway. 

setTimeout(function() {
    var
    console.log("asynchronous stuff has finished");
    jsbox.test();            
}, 2000);
